1. Детальный архитектурный контекст
Проект представляет собой распределенную систему с четким разделением ответственности, построенную вокруг концепции User-Managed Access (UMA) через Keycloak и BPMN 2.0 процессов через SpiffWorkflow.
1.1. Контекстная диаграмма (C4 — Level 1)
    Пользователь (Browser): Взаимодействует с SPA (React).
    SPA (React + TS + Vite + Ant Design): Клиентское приложение. Отвечает за UI/UX, маршрутизацию на стороне клиента, управление состоянием (например, Redux Toolkit/Zustand) и взаимодействие с API бэкенда.
    Backend Core (FastAPI): Ядро системы.
        Выступает в роли Resource Server для SPA.
        Выступает в роли API Gateway для внешних сервисов (Spiff, Novu), скрывая их от фронтенда.
        Содержит бизнес-логику, не связанную напрямую с процессами.
        Управляет данными приложения (PostgreSQL).
    PostgreSQL 18.1: Основная БД бэкенда.
    Keycloak: Сервер авторизации (Identity Provider).
        Управляет пользователями, ролями, группами.
        Выдает JWT токены.
        Настроен как SSO.
    SpiffWorkflow (Отдельный сервис): Специализированный движок BPMN.
        Хранит определения процессов (BPMN, DMN).
        Выполняет экземпляры процессов.
        Предоставляет REST API для запуска процессов и выполнения пользовательских задач (User Tasks).
    Novu (Cloud или Self-hosted): Сервис уведомлений.
        Агрегирует каналы (Email, SMS, In-App, Push).
        Принимает запросы от Backend Core.
        Debian 13 (Server): Хост-система.
1.2. Поток данных (Sequence)
    Аутентификация:
        SPA редиректит на Keycloak (OIDC Authorization Code Flow с PKCE).
        После логина Keycloak редиректит обратно на SPA с кодом.
        SPA обменивает код на токены (Access, Refresh) через бэкенд или напрямую (рекомендуется через бэкенд для безопасности).
        API Request:
        SPA отправляет запрос с Bearer JWT на https://project.ukituki.tech/api/v1/....
        FastAPI валидирует JWT (проверяет подпись ключами Keycloak, срок действия, aud).
        FastAPI извлекает роли/пользователя из токена.
    Запуск процесса:
        Пользователь инициирует процесс.
        FastAPI вызывает POST /v1/processes у SpiffWorkflow.
        SpiffWorkflow возвращает process_instance_id.
        FastAPI сохраняет связь user_id ↔ process_instance_id в своей БД.
    Выполнение User Task:
        SpiffWorkflow создает задачу и ждет колбэка.
        SPA запрашивает у FastAPI список доступных задач.
        Пользователь заполняет форму (Ant Design).
        SPA отправляет данные в FastAPI.
        FastAPI вызывает SpiffWorkflow для завершения задачи (POST /v1/tasks/{id}/complete).
    Уведомления:
        В процессе выполнения задачи (или по триггеру в БД), FastAPI отправляет запрос в Novu (trigger).
        Novu обрабатывает шаблоны и отправляет уведомление пользователю.
2. Верхнеуровневое задание (Project Brief)
Цель: Разработать и развернуть на сервере project.ukituki.tech масштабируемое веб-приложение с микросервисной архитектурой, использующее централизованную аутентификацию (Keycloak), оркестрацию бизнес-процессов (SpiffWorkflow) и гибкую систему уведомлений (Novu).
    Функциональные требования (High-level):
    Пользователь может зарегистрироваться/войти через Keycloak.
    Пользователь может инициировать бизнес-процесс (BPMN), загруженный в SpiffWorkflow.
    Пользователь видит свой дашборд со списком активных процессов и ожидающих его задач (User Tasks).
    Пользователь может выполнять шаги процесса через формы, сгенерированные на основе схемы данных задачи.
    Система уведомляет пользователя о назначении новой задачи (in-app уведомление через Novu).
    Администратор может загружать новые BPMN-диаграммы через админ-панель (или API).
Нефункциональные требования:
    Безопасность: Все API-эндпоинты (кроме healthchecks) защищены JWT, ролевая модель основана на Keycloak.
    Тестирование: Покрытие кода бэкенда >80% (unit + integration). Наличие smoke-тестов для инфраструктуры.
    DevOps: Один скрипт deploy.sh для развертывания всех сервисов на целевом сервере.
    Миграции: Автоматическое применение миграций БД (Alembic) при деплое.
